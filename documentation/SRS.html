Software Requirements Specification

1.                  Introduction

1.1               Purpose

This SRS describes the specifications for the social trend board – strebo. The document will provide an overview over the „Use Case Diagram“ and explains the used technologies.

1.2               Scope

strebo is a web-based application that combines the trends of various social media networks on one single page. Furthermore the user will be able to connect it's own social media accounts to get access to a personal dashboard, to view it's personally relevant content.

1.3               Definitions, Acronyms and Abbreviations

strebo
social trend board

Social Media Network
also Social Network or Network
plural: Social Media Networks
A virtual community where people can create, share or exchange information, ideas and pictures/videos with others.

1.4               References

GitHub

Heroku-Web-App

1.5               Overview

First, this document will provide an overall description. After that the requirements will be specified and at the end there will be supporting information.

2.                  Overall Description

Strebo is a web based application for showing trends of all the supported social networks and aims to reduce the complexity of staying up to date on ones social networks.
To achieve this goal, strebo provides a clear and uniform way to get all the important news of the single networks and give the user the abilities to interact with all of them at once. For example the user is able to search in all of the supported networks for posts specified by entered keywords.

One of the two main functions of strebo is the trend board. The trend board gives the user an overview of what actually is going on in the single networks. The other main function of strebo is the personal dash board. After an successful login procedure to the social networks of the user, all the newsfeeds of the users networks get embedded in his personal dashboard.

3.                  Specific Requirements

strebo requires a modern web browser for full functionality. In addition there are strong dependencies to several APIs. To get information from social media platforms we use APIs like the Twitter API or Facebook API. The support of APIs and their functions are very different, so the integration of these APIs will be also very different and maybe it is also necessary to scrape data from websites.

Search for specific posts and news in the supported Social Networks. Does not require the user to be connected with Social Media Networks.

3.1.5               Connect to Supported Social Networks.

This function uses the different APIs provided by the Social Media Networks.
After strebo is connected via the APIs, the Trend Board uses the gathered information from the Networks to display them. The connection to the APIs is also used to grant the users the ability to connect to their own Social Media accounts.
The following Social Media Networks are planned:


  • Connect to YouTube
  • Connect to Twitter
  • Connect to Instagram
  • Connect to Tumblr
  • Connect to Facebook
  • Connect to other Social Networks

The use case "Connect to other Social Networks" is a placeholder for more Social Networks that might be supported in the future.

3.1.6               Switch between different Views

There will be different Views available. With "View" is meant how the recent trends and posts are displayed on the screen.
The user will be able to change between the provided Views.

3.2               Usability

strebo aims to be easy understandable, so that there is only a small demand for a tutorial.
The trend board is going to update automatically, either after a specified period of time or after the happening of a specific event, namely a new hashtag starts to be trendy.
The login function, which allows users to connect to different Social Media Platforms, will take the user very few moments to enter his or her login-information. After that strebo will automatically take the latest posts and display them in the personal dashboard.

3.3               Reliability

3.3.1               Availability

It is very likely that strebo will be available 24 hours a day (server issues can happen, so we can't guarantee that), so the users are able to use it whenever they like to do so!

3.3.2               Mean Time Between Failures

to be determined

3.3.3               Mean Time To Repair

The mean time to repair depends on the reason for the failure and the time it occurs.
If the failure was caused by an update it will take roughly an hour to deploy an earlier version.
If the failure was caused by a server crash or due to the unavailability of one or more APIs, the mean time to repair depends on how fast the providers are able to restore their systems.

3.3.4               Accuracy

There will be several test types to ensure that strebo is working as intended to provide a high accuracy.

3.3.5               Bugs

There are several types of bugs, that are categorized here.

Minor Bug
A minor bug has no influence on the functions of strebo, but it can cause the system to be slightly slower, what might be noticeable in longer loading time.
Significant Bug
A significant bug has influence on the usability of strebo. This means that the users are not able to access the full functionality of strebo.
Critical Bug
A critical bug causes the program to be terminated. strebo will not be available until the bug is fixed.
